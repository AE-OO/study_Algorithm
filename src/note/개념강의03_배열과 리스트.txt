개념강의 03. 배열과 리스트

◆ 배열
- 메모리의 연속적인 공간에 값이 채워져 있는 형태
- 인덱스를 통해 값을 바로 참조할 수 있음
- 새로운 값을 삽입하거나 특정 인덱스의 값을 삭제하기 어려움
- 크기를 한번 정하면 늘리거나 줄일 수 없음

=> 데이터의 수가 정해져 있는 경우, 데이터에 접근하는 경우가 많은 경우 사용하는 것이 좋음

◆ 리스트
- 값과 포인터를 묶은 노드를 포인터로 연결한 자료구조 (포인터 : 다음 노드를 가리킴)
- 인덱스가 없어서 제일 첫 포인터부터 순서대로 접근해야 함 => 느림
- 데이터를 삽입하거나 삭제하는 속도가 빠름
- 크기가 정해져 있지 않음

=> 크기가 변하는 데이터를 다루는 경우, 데이터의 삽입, 삭제가 많은 경우 사용하는 것이 좋음
